# OLWatch 프로젝트 기획서

**YouTube 크리에이터 시리즈 정주행 플랫폼**

---

## 1. 프로젝트 개요

### 1.1 서비스명

**OLWatch (올와치)**

- 의미: All + Watch = 모든 영상을 보다
- 발음: [올와치]

### 1.2 서비스 개념

YouTube 개인 크리에이터가 자신의 영상들로 시리즈 정주행을 유도할 수 있는 서드파티 웹 서비스입니다. Netflix와 같은 시청 경험을 제공하여, 시청자가 특정 크리에이터의 콘텐츠에 집중하며 연속 시청할 수 있도록 지원합니다.

### 1.3 핵심 가치 제안

- **크리에이터 관점**: 자신의 콘텐츠로 시청자를 유지하고 시리즈 완주율을 높임
- **시청자 관점**: YouTube 알고리즘에 방해받지 않고 원하는 크리에이터의 시리즈를 집중해서 정주행
- **차별점**: 썸네일 중심의 YouTube와 달리, 설명(description)과 순서를 크리에이터가 직접 조정하여 Netflix처럼 구성

---

## 2. 문제 인식 및 솔루션

### 2.1 현재 YouTube의 문제점

- **알고리즘 중심 추천**: 시청자를 플랫폼에 묶어두기 위해 다양한 콘텐츠를 추천하며, 특정 크리에이터의 시리즈 완주를 방해
- **썸네일 의존도**: 영상 선택이 주로 썸네일에 의존하며, 콘텐츠의 맥락과 설명을 충분히 전달하지 못함
- **연속 시청 어려움**: 재생목록 기능은 있으나, 시청 경험이 분산되고 다음 영상으로의 전환이 자연스럽지 않음

### 2.2 OLWatch의 솔루션

- **맞춤형 타이틀 및 설명**: 크리에이터가 시리즈용 타이틀과 디스크립션을 수정하여 맥락 제공
- **영상 순서 조정**: 크리에이터가 시리즈의 이야기 흐름에 맞게 영상 순서를 재배열
- **끊김없는 정주행**: 영상 종료 시 자동으로 다음 영상으로 이동하여 Netflix 같은 연속 시청 경험 제공
- **집중된 시청 환경**: 알고리즘 추천 없이 해당 시리즈에만 집중할 수 있는 UI

## 3. 프로토타입 범위 및 목표

### 3.1 프로토타입 목적

- **타겟 사용자**: 팀원 및 초기 고객에게 서비스 컨셉 시연
- **부차적 활용**: VC 미팅 시 스몰톡 소재로 활용 가능
- **핵심 목표**: 서비스의 핵심 기능과 사용자 경험을 간단하게 검증

### 3.2 구현 범위

#### 페이지 구성 (총 2페이지)

1. **시리즈 재생목록 페이지**
   - 시리즈 전체 정보 및 영상 목록 표시

2. **영상 재생 페이지**
   - 중앙 고정 영상 플레이어와 하단에 '다음 영상' 버튼, '목록으로' 버튼 배치

#### 데이터 및 콘텐츠

- **단일 시리즈**: 운영자 본인의 YouTube 채널 기반 하나의 시리즈만 구현
- **정적 데이터**: 수동으로 입력한 데이터 사용 (백엔드 없이 프론트엔드만 구현)
- **영상 수**: 2-3개 영상으로 구성

#### 제외 사항

- 메인 페이지 (여러 시리즈 선택 기능)
- 사용자 인증 및 계정 시스템
- 백엔드 API (시간이 허용되면 설계 테스트 정도만 진행)
- 크리에이터 관리자 페이지

## 4. 기술 스택

### 4.1 프론트엔드

- **React**: UI 구현 (Netflix가 사용하는 기술 스택과 동일)
- **YouTube IFrame Player API**: 영상 임베드 및 자동 재생 제어

### 4.2 백엔드 (선택 사항)

프로토타입 단계에서는 구현하지 않으나, 시간이 허용될 경우 백엔드 설계를 테스트할 수 있습니다.

- **Node.js + Express** 또는 **NestJS**
- 크리에이터 데이터 관리 및 시리즈 메타데이터 저장

---

## 5. 주요 기능 명세

### 5.1 시리즈 재생목록 페이지

#### UI 구조 (상단에서 하단 순서)

1. **시리즈 썸네일**
   - 시리즈를 대표하는 커버 이미지

2. **타이틀**
   - 시리즈 제목

3. **제작자**
   - 크리에이터 이름/채널명

4. **비디오 수 & 뷰 수**
   - 예: "3개의 영상 · 총 15.2만 조회수"

5. **액션 버튼**
   - **첫화부터 보기 버튼**: 첫 번째 영상 재생 페이지로 이동
   - **공유 버튼**: 현재 페이지 링크 복사

6. **영상 리스트**
   - 각 영상 카드 구조:
     - 타이틀 (상단)
     - 디스크립션 (하단)
     - 재생 시간
   - 세로 스크롤 가능한 리스트 형태

7. **푸터**
   - 사이트 정보 및 링크

#### 인터랙션

- 영상 카드 클릭 시 해당 영상의 재생 페이지로 이동
- "첫화부터 보기" 버튼 클릭 시 첫 번째 영상 재생 페이지로 이동
- "공유" 버튼 클릭 시 현재 페이지 URL 클립보드 복사

### 5.2 영상 재생 페이지

#### UI 요소

- **중앙 고정 영상 플레이어**: YouTube iframe embed 사용
- **영상 정보**: 수정된 타이틀 및 디스크립션 표시
- **내비게이션 버튼**:
  - '다음 영상' 버튼
  - '목록으로' 버튼

#### 자동 재생 기능

- **영상 종료 감지**: YouTube IFrame Player API의 onStateChange 이벤트 활용
- **자동 이동**: 영상 종료 시 자동으로 다음 영상 페이지로 이동
- **자동 재생**: autoplay=1 파라미터로 다음 영상 자동 재생
- **사운드 유지**: 사용자의 초기 클릭(인터랙션) 이후 자동 재생되는 영상은 음소거 없이 정상 재생

#### 데이터 구조 예시

각 영상은 다음과 같은 정보를 포함합니다:

- YouTube 영상 ID
- 수정된 타이틀
- 수정된 디스크립션
- 재생 시간
- (선택 사항) 썸네일 URL

## 7. 향후 확장 계획

프로토타입 검증 후 다음 단계로 확장 가능한 기능들입니다.

### 7.1 크리에이터 기능

- 크리에이터 대시보드: 시리즈 생성 및 관리
- 영상 메타데이터 편집: 타이틀, 디스크립션, 순서 조정
- 분석 기능: 시리즈 완주율, 시청 패턴 등

### 7.2 사용자 기능

- 여러 크리에이터의 시리즈 탐색
- 시청 기록 및 북마크
- 추천 시스템 (크리에이터 기반)

### 7.3 기술적 확장

- 백엔드 API 구축 (Node.js/NestJS)
- 데이터베이스 연동 (PostgreSQL/MongoDB)
- YouTube Data API 통합 (자동 메타데이터 가져오기)
- 사용자 인증 시스템 (OAuth)
- 반응형 디자인 (모바일 최적화)

## 8. 프로토타입 성공 지표

프로토타입의 성공 여부는 다음 기준으로 평가합니다.

### 8.1 기능적 완성도

- 시리즈 재생목록 페이지가 정상적으로 렌더링됨
- 영상 재생 페이지로의 전환이 원활함
- 영상 종료 후 자동으로 다음 영상으로 이동
- 소리와 함께 자동 재생이 작동함
- "첫화부터 보기" 및 "공유" 버튼이 정상 작동함

### 8.2 사용자 피드백

- 팀원 및 초기 고객이 서비스 컨셉을 이해함
- Netflix 스타일의 정주행 경험이 긍정적으로 평가됨
- 추가로 원하는 기능에 대한 피드백 수집

### 8.3 개발 효율성

- 계획된 시간(4-7시간) 내에 프로토타입 완성
- 코드 구조가 향후 확장에 용이하도록 설계됨

## 9. 결론

OLWatch는 YouTube 크리에이터와 시청자 모두에게 집중된 정주행 경험을 제공하는 혁신적인 서비스입니다. 프로토타입을 통해 핵심 기능을 검증하고 사용자 반응을 수집한 후, 단계적으로 기능을 확장하여 완전한 플랫폼으로 발전시킬 계획입니다.

이 프로젝트는 YouTube의 알고리즘 중심 추천 시스템에 대한 대안을 제시하며, 크리에이터가 자신의 콘텐츠를 더 효과적으로 전달하고 시청자가 원하는 콘텐츠에 집중할 수 있는 환경을 만들어줍니다.
